# Jira 개인용 퀵 매니저 웹앱 요구사항

## 1. 개요

Jira 기본 UI의 느린 필터링과 불편한 화면을 보완하기 위해, 개인 중심의 빠른 티켓 조회 및 생성 웹앱을 만든다.

이 앱의 핵심 가치는 아래 두 가지다.

- 내 티켓을 빠르게 본다.
- 반복되는 패턴으로 티켓을 빠르게 생성한다.

배포는 React 정적 빌드 후 GitHub Pages로 진행하며, 브라우저에서 직접 Jira API를 호출한다. 설정값과 캐시 데이터는 IndexedDB에 저장한다.

## 2. 제품 방향

### 2.1 제품 성격

- 개인 우선 도구
- 사내 배포 가능
- 기본 UX는 "내 업무 생산성 향상" 중심

이 앱은 Jira 전체 기능을 대체하지 않는다. 아래 영역에 집중한다.

- 내 티켓 빠른 조회
- 나를 언급한 이슈 확인
- 자주 쓰는 Epic 기반 빠른 생성
- 최근 사용 패턴 기반 추천

### 2.2 목표

- Jira보다 빠르게 개인 업무 티켓을 조회할 수 있어야 한다.
- 반복 생성 패턴을 최소 입력으로 처리할 수 있어야 한다.
- 캐시 우선 UX로 빠른 체감을 제공해야 한다.

## 3. 핵심 기능 요약

### 3.1 설정

최초 1회 사용자가 아래 정보를 입력한다.

| 항목 | 설명 |
| --- | --- |
| Jira 도메인 prefix | 입력 UI는 prefix만 받고, 실제 주소는 `https://{prefix}.atlassian.net` |
| 이메일 | Jira 계정 이메일 |
| API 토큰 | Jira API 호출용 토큰 |

기능 요구사항:

- 저장 위치는 IndexedDB
- 앱 시작 시 자동 복원
- 설정 화면에서 수정/삭제 가능
- 연결 테스트 버튼 제공
- 수동 동기화 버튼 제공
- 마지막 동기화 시각 표시

### 3.2 기준 데이터 동기화

조회 및 생성에 필요한 기준 데이터를 캐시한다.

동기화 대상:

- 프로젝트 리스트
- 사용자 리스트
- 자주 사용하는 Epic 관련 정보
- 최근 생성 시 사용한 메타 정보

동기화 정책:

- 하루 1회 자동 동기화
- 사용자가 원할 때 수동 동기화 가능
- 동기화 실패 시 기존 캐시 유지
- 저장소는 IndexedDB 사용

### 3.3 메인 대시보드

앱 첫 화면이며, 개인이 바로 확인해야 하는 티켓을 빠르게 보여준다.

주요 섹션:

- 내 티켓
- 상태별 티켓
- 나를 언급한 이슈
- 최근 생성한 티켓
- 빠른 생성 영역

### 3.4 내 티켓 조회

기본 조회 기준:

- 나에게 할당된 티켓
- 내가 reporter인 티켓
- 내가 watcher인 티켓
- 나를 mention한 이슈
- 내가 최근에 본 티켓
- 내가 최근에 생성한 티켓

보기 방식:

- 프로젝트별 그룹
- 상태별 그룹
- Epic별 그룹
- 리스트형 / 카드형 전환

추천 기본 탭:

- 전체
- 내 할당
- 진행중
- 대기중
- 완료
- 언급됨

### 3.5 상태별 보기

내 티켓을 상태별로 빠르게 나누어 확인한다.

예시 상태:

- To Do
- In Progress
- Review
- Done

주의사항:

- 실제 Jira 상태명은 프로젝트마다 다를 수 있다.
- 내부적으로는 Jira 상태명을 그대로 사용한다.
- UI에서는 그룹화 가능하게 설계한다.

각 티켓 카드 최소 표시 항목:

- Key
- Summary
- Project
- Status
- Assignee
- Epic
- Updated at

카드 액션:

- Jira에서 열기
- 복사
- 빠른 필터 적용

### 3.6 나를 언급한 이슈 보기

중요 기능으로, 별도 탭 또는 대시보드 섹션으로 제공한다.

표시 정보:

- 이슈 key
- 제목
- 프로젝트
- 최근 업데이트 시각
- 최근 언급 활동 여부

제약사항:

- 댓글 작성/조회 기능은 없어도 된다.
- 조회와 Jira 링크 이동만 지원하면 된다.

### 3.7 티켓 빠른 생성

이 앱의 핵심 가치 중 하나다.

입력 항목:

- 프로젝트 선택
- 이슈 타입 선택
- 담당자 선택
- 제목
- 내용
- Epic 선택

생성 후 액션:

- 생성 성공 알림
- 방금 만든 티켓 브라우저로 열기
- 동일한 값으로 다시 생성

### 3.8 Epic 기능

Epic 기능은 매우 중요하며 아래 요구사항을 포함한다.

#### 3.8.1 Epic 지정 생성

- 티켓 생성 시 Epic을 지정할 수 있어야 한다.

#### 3.8.2 최근 사용 Epic 추천

추천 기준 예시:

- 최근 사용 순
- 최근 30일 사용 횟수
- 프로젝트별 최근 사용 Epic

#### 3.8.3 Epic별 티켓 생성기

반복 업무를 위해 Epic 단위 빠른 생성 기능을 제공한다.

예시 흐름:

1. 특정 Epic 선택
2. 미리 정한 프로젝트/담당자/이슈타입 조합 추천
3. 최소 입력으로 티켓 생성

즉, "Epic을 고르면 반복 티켓을 빠르게 생성하는 모드"가 필요하다.

### 3.9 최근 사용 기반 추천

입력 속도 향상을 위한 기능이다.

추천 대상:

- 최근 선택한 프로젝트
- 최근 선택한 담당자
- 최근 사용한 Epic
- 최근 사용한 이슈 타입

동작 방식:

- 폼 진입 시 기본값 또는 추천값으로 노출

### 3.10 최근 생성 티켓

방금 만든 티켓 또는 최근 생성 티켓을 별도로 보여준다.

요구사항:

- 최근 생성 10개 표시
- 클릭 시 Jira에서 열기
- 같은 프로젝트/에픽으로 다시 생성 가능

## 4. 화면 구성

### 4.1 전체 레이아웃

좌측 사이드바 메뉴:

- 대시보드
- 내 티켓
- 상태별 보기
- 언급됨
- 빠른 생성
- Epic 생성기
- 최근 생성
- 설정

상단 바:

- 전역 검색 입력
- 수동 동기화 버튼
- 마지막 동기화 시각
- 설정 바로가기

메인 콘텐츠:

- 선택한 메뉴에 따라 내용 표시

### 4.2 대시보드

상단:

- 빠른 생성 버튼
- 최근 Epic 버튼
- 최근 프로젝트 버튼

본문:

- 내 할당 티켓 요약
- 상태별 카운트
- 언급된 이슈
- 최근 생성 티켓
- 프로젝트별 티켓 요약

### 4.3 내 티켓 화면

상단 필터:

- 프로젝트 필터
- 상태 필터
- 담당자 필터
- Epic 필터
- 내 것만 토글
- 언급된 것만 토글

본문:

- 리스트형 / 카드형 전환
- 기본은 리스트형
- 그룹 기준 선택: 프로젝트별 / 상태별 / Epic별

### 4.4 상태별 보기 화면

칸반 스타일에 가깝게 단순 구성한다.

구성:

- 상태별 컬럼 자동 생성
- 각 컬럼 안에 내 티켓 카드 표시

카드 클릭 시:

- 상세 패널 열기 또는 Jira 열기

앱 내부 상세 조회는 최소화해도 된다. 핵심은 빠르게 보고 Jira로 이동하는 것이다.

### 4.5 빠른 생성 화면

상단:

- 최근 사용 Epic 추천 칩
- 최근 프로젝트 추천 칩
- 최근 담당자 추천 칩

폼:

- 프로젝트
- 이슈 타입
- 담당자
- Epic
- 제목
- 내용

하단 버튼:

- 생성
- 생성 후 티켓 열기
- 폼 초기화

### 4.6 Epic 생성기 화면

좌측:

- 자주 쓰는 Epic 리스트
- 최근 사용 Epic
- 프로젝트별 Epic 필터

우측:

- 선택된 Epic 기준 생성 폼

기능:

- 해당 Epic 자동 세팅
- 프로젝트/담당자/이슈타입 추천
- 제목 템플릿 입력 가능
- 반복적으로 같은 Epic에 티켓 생성 가능

### 4.7 설정 화면

연결 설정 카드:

- 도메인 prefix 입력 + `.atlassian.net` 고정 표시
- 이메일
- API 토큰
- 저장 버튼
- 연결 테스트 버튼

데이터 관리 카드:

- 프로젝트/사용자 동기화 버튼
- 캐시 초기화 버튼
- 마지막 동기화 시각 표시

사용성 설정 카드:

- 최근 Epic 추천 개수
- 최근 프로젝트 개수
- 기본 화면 설정
- 기본 그룹 기준 설정

## 5. 데이터 저장 정책

저장소는 IndexedDB를 사용한다.

저장 대상:

- 설정값
- `domainPrefix`
- `email`
- `apiToken`
- 프로젝트 캐시
- 사용자 캐시
- 최근 사용 Epic
- 최근 사용 프로젝트
- 최근 사용 담당자
- 최근 생성 티켓
- 마지막 동기화 시각
- 사용자 UI 설정값

저장 원칙:

- `localStorage` 사용 금지
- 대용량 목록 캐시는 IndexedDB 사용
- 민감 정보도 IndexedDB 저장 가능하나 UI에서 명확히 안내
- 캐시 실패 시 graceful fallback 제공

## 6. 기술 원칙

필수 원칙:

- React 기반
- 정적 빌드 후 GitHub Pages 배포
- 브라우저에서 직접 Jira API 호출
- IndexedDB 사용
- TypeScript 사용 권장
- 함수형 컴포넌트 사용
- 외부 CDN 사용 금지
- 외부 `script` 태그 사용 금지
- 상태 관리 과도하게 키우지 말 것
- 먼저 동작하는 MVP 구현 후 고도화

구현 원칙:

- MDN 문서 참조 금지
- 외부 UI 라이브러리 없이 기본 React/CSS로 구현
- 스타일은 단순하고 빠르게
- Jira 기본 UI보다 빠른 체감이 목표
- 지나친 추상화 금지
- 리팩토링보다 기능 완성을 우선
- 각 기능은 독립적으로 테스트 가능하게 구현

## 7. 보안 및 운영 원칙

- 어떠한 비밀값도 코드에 하드코딩하지 않는다.
- Jira 도메인은 사용자 입력값으로만 저장한다.
- API 토큰도 사용자 입력값으로만 받는다.
- 토큰/도메인/이메일은 배포 파일에 포함되면 안 된다.
- 외부 라이브러리는 최소화한다.
- 댓글 작성 기능은 구현하지 않는다.
- Jira 데이터 수정 기능은 최소화한다.
- 우선순위는 조회 + 생성이다.

- 프로젝트 목록 조회
- 사용자 목록 조회
- 내 티켓 조회
- 상태별 티켓 조회
- 언급된 이슈 조회
- Epic 관련 조회
- 최근 업데이트 티켓 조회

### 8.2 생성

- 일반 티켓 생성
- Epic 지정 티켓 생성
- 생성 후 생성된 티켓 key/url 반환
- 브라우저에서 해당 티켓 열기 지원

### 8.3 제외 범위

- 댓글 작성/수정
- 첨부파일

## 9. 추천 UX 디테일

빠른 입력 UX:

- 최근 사용값 자동 추천
- 검색형 드롭다운
- 최근 사용 Epic 상단 고정
- 생성 직후 "같은 Epic으로 하나 더 만들기" 버튼
- 프로젝트 변경 시 담당자/Epic 추천도 프로젝트 기준으로 갱신

속도 UX:

- 최초 진입 시 캐시 우선 표시
- 백그라운드 동기화 후 갱신
- 큰 목록은 가상 스크롤 없이도 렌더링 최적화 고려
- 불필요한 전체 재조회 금지

- 설정 저장
- 프로젝트/사용자 동기화
- 내 티켓 조회
- 상태별 보기
- 언급된 이슈 보기
- 티켓 생성
- Epic 지정 생성
- 최근 사용 Epic 추천
- 최근 생성 티켓 보기
- 생성 후 Jira에서 열기

## 11. 2차 확장 후보

- 제목 템플릿 저장
- Epic별 프리셋 생성기
- 프로젝트별 기본 담당자 프리셋
- 다크모드
- 단축키
- 오프라인 캐시 강화
- 내 티켓 커스텀 그룹 저장

## 12. 한 줄 요약

React 기반 정적 웹앱을 GitHub Pages에 배포하고, Jira 도메인/email/API 토큰은 설정 화면에서 입력받아 IndexedDB에 저장한다. 프로젝트/사용자 목록은 하루 1회 자동 또는 수동 동기화하며, 핵심 기능은 내 티켓 빠른 조회, 상태별 보기, 나를 언급한 이슈 보기, Epic 지정 포함 퀵 티켓 생성, 최근 사용 Epic 추천, Epic별 반복 티켓 생성, 최근 생성 티켓 열기다.
